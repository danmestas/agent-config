# Software Philosophy

Adversarial proofing for code and plans. Six complementary lenses for writing better software.

## Skills

| Skill | When to use |
|-------|-------------|
| **tigerstyle** | Safety-critical code, systems programming, assertion-heavy development |
| **hipp** | Designing simple, reliable, self-contained modules (SQLite philosophy) |
| **ousterhout** | Module/API design, reducing complexity, information hiding |
| **norman** | UI design, usability review, interaction patterns |
| **dx-audit** | Evaluating developer/user experience, scoring ergonomics |
| **idiomatic-go** | Writing, reviewing, or refactoring Go code |

## Install

```bash
claude plugin marketplace add danmestas/agent-skills
claude plugin install software-philosophy
```
